# Test Report Format Reference

How actionbook-web-test generates visual HTML reports using **json-ui** (`@actionbookdev/json-ui`).

## Overview

After test execution, the agent constructs a json-ui JSON document from collected test results, then renders it to a standalone HTML file. The HTML is self-contained (inline CSS, embedded screenshots) and works offline.

## Report Structure

A test report is a json-ui `Report` node with this layout:

```
Report
├── BrandHeader              — Report title and branding
├── Section: Summary         — MetricsGrid with pass/fail/skip/duration
├── Section: Environment     — DefinitionList with browser, URL, profile info
├── Section: Results         — Table with per-test status overview
├── Section: <test-name> (×N, one per test, collapsible)
│   ├── Callout              — Error details (FAIL tests only)
│   ├── ContributionList     — Step-by-step execution log with PASS/FAIL/SKIP badges
│   └── Image                — Screenshot at end of test (base64 data URL, ALL tests)
├── Section: Console         — Callout with JS errors (if any)
└── BrandFooter              — Timestamp and attribution
```

**Every test gets its own collapsible section** with a `ContributionList` and `Image`, regardless of PASS/FAIL/SKIP status. The section `icon` reflects the outcome: `"check"` for PASS, `"warning"` for FAIL, `"skip"` for SKIP.

## Component Construction Guide

### 1. Summary Metrics

Build `MetricsGrid` from the test summary:

```typescript
// Pseudocode — agent constructs this mentally
const metricsGrid = {
  type: "MetricsGrid",
  props: {
    cols: 5,
    metrics: [
      {
        label: "Total",
        value: String(summary.total),
        icon: "list"
      },
      {
        label: "Passed",
        value: String(summary.passed),
        trend: summary.passed > 0 ? "up" : undefined,
        icon: "check"
      },
      {
        label: "Failed",
        value: String(summary.failed),
        trend: summary.failed > 0 ? "down" : undefined,
        icon: "warning"
      },
      {
        label: "Skipped",
        value: String(summary.skipped),
        icon: "skip"
      },
      {
        label: "Duration",
        value: formatDuration(summary.duration),  // e.g. "12.4s"
        icon: "clock"
      }
    ]
  }
};
```

**Trend indicators:**
- `"up"` — green arrow, use for passed count
- `"down"` — red arrow, use for failed count
- omit — neutral, use for total/skipped/duration

### 2. Results Table

Build `Table` rows from each test result:

```typescript
const resultsTable = {
  type: "Table",
  props: {
    columns: [
      { key: "status", label: "Status" },
      { key: "name", label: "Test" },
      { key: "steps", label: "Steps" },
      { key: "assertions", label: "Assertions" },
      { key: "duration", label: "Duration" },
      { key: "tags", label: "Tags" }
    ],
    rows: results.map(r => ({
      status: r.status === "passed" ? "PASS" :
              r.status === "failed" ? "FAIL" : "SKIP",
      name: r.name,
      steps: `${r.steps.passed}/${r.steps.total}`,
      assertions: `${r.assertions.passed}/${r.assertions.total}`,
      duration: formatDuration(r.duration),
      tags: r.tags?.join(", ") ?? ""
    })),
    striped: true
  }
};
```

### 3. Per-Test Detail Sections

Generate one collapsible `Section` for **every test** (PASS, FAIL, or SKIP). Each section contains a `ContributionList` of steps and an `Image` with the test's final screenshot.

```typescript
const testSection = {
  type: "Section",
  props: {
    title: `${result.name} — Step Details`,
    icon: result.status === "passed" ? "check" :
          result.status === "failed" ? "warning" : "skip",
    collapsible: true
  },
  children: [
    // Error callout — ONLY for failed tests
    ...(result.status === "failed" ? [{
      type: "Callout",
      props: {
        type: "important",
        title: `Step ${failedStep.index}: ${failedStep.name}`,
        content: [
          `**Error:** ${failedStep.error}`,
          failedStep.selector ? `**Selector:** \`${failedStep.selector}\`` : null,
          failedStep.expected ? `**Expected:** ${failedStep.expected}` : null,
          failedStep.actual ? `**Actual:** ${failedStep.actual}` : null
        ].filter(Boolean).join("\n\n")
      }
    }] : []),
    // Step-by-step execution log — ALL tests
    {
      type: "ContributionList",
      props: {
        numbered: true,
        items: result.steps.map(step => ({
          title: step.name,
          badge: step.status === "passed" ? "PASS" :
                 step.status === "failed" ? "FAIL" : "SKIP",
          description: step.cli_command
            ? `\`${step.cli_command}\` (${formatDuration(step.duration)})`
            : `Skipped`
        }))
      }
    },
    // Final screenshot — ALL tests (MUST be base64 data URL)
    {
      type: "Image",
      props: {
        src: result.screenshotBase64,  // "data:image/png;base64,..."
        alt: `${result.name} — ${result.status} state`,
        caption: result.status === "failed"
          ? `Page state at point of failure (${failedStep.name})`
          : `Page state after test completed (${result.status.toUpperCase()})`
      }
    }
  ]
};
```

### 4. Console Errors Section

If JS errors were captured during the test run:

```typescript
const consoleSection = {
  type: "Section",
  props: { title: "Console Errors", icon: "warning", collapsible: true },
  children: consoleErrors.map(error => ({
    type: "Callout",
    props: {
      type: "warning",
      title: error.source ?? "JavaScript Error",
      content: `\`\`\`\n${error.message}\n\`\`\``
    }
  }))
};
```

Only include this section if there are console errors to report.

### 5. Environment Info

```typescript
const envSection = {
  type: "Section",
  props: { title: "Environment", icon: "settings", collapsible: true },
  children: [
    {
      type: "DefinitionList",
      props: {
        items: [
          { term: "Timestamp", definition: new Date().toISOString() },
          { term: "Browser", definition: "Chromium 125.0.6422.0 (headless)" },
          { term: "Viewport", definition: "1280x720" },
          { term: "Profile", definition: setup.profile ?? "default" },
          { term: "Mode", definition: "extension" },  // or "cdp"
          { term: "Target URL", definition: workflow.url },
          { term: "Tags", definition: workflow.tags.join(", ") }
        ]
      }
    }
  ]
};
```

## Rendering

The json-ui package is `@actionbookdev/json-ui` on npm. Use `npx` — no global install needed:

```bash
# Write the report JSON to a file first, then render:
npx @actionbookdev/json-ui render test-report.json -o test-report.html

# Render without auto-opening browser
npx @actionbookdev/json-ui render test-report.json -o test-report.html --no-open

# Pipe from stdin
cat test-report.json | npx @actionbookdev/json-ui render - -o test-report.html
```

**IMPORTANT**: Always write the JSON to a file first, then render with `npx @actionbookdev/json-ui`. Do NOT attempt to generate raw HTML directly — json-ui handles all styling, theming, dark mode, and responsive layout.

## Duration Formatting

| Raw (ms) | Formatted |
|----------|-----------|
| 150 | `0.2s` |
| 3200 | `3.2s` |
| 12400 | `12.4s` |
| 65000 | `1m 5s` |
| 180000 | `3m 0s` |

## Status Badge Colors

| Status | Display | Color Intent |
|--------|---------|-------------|
| `passed` | `PASS` | green (success) |
| `failed` | `FAIL` | red (danger) |
| `skipped` | `SKIP` | gray (muted) |
| `error` | `ERROR` | orange (warning) |
